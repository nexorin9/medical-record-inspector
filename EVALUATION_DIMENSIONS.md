# 评估维度说明

## 概述

Medical Record Inspector 从四个维度评估病历质量：

1. **完整性 (Completeness)**: 病历基本要素的覆盖程度
2. **逻辑性 (Logicality)**: 医学推理和内容连贯性
3. **及时性 (Timeliness)**: 时间记录的准确性和及时性
4. **规范性 (Normativity)**: 书写格式和术语的规范性

每个维度评分范围为 0-100 分，最终得分为四个维度的加权平均。

## 1. 完整性 (Completeness)

完整性评估检查病历是否包含法律法规和临床实践要求的基本要素。

### 评估指标

| 指标 | 权重 | 检查内容 |
|------|------|----------|
| 主诉 | 20% | 是否清晰描述主要症状和持续时间 |
| 现病史 | 25% | 起病情况、症状特点、诊疗经过等 |
| 既往史 | 15% | 既往疾病、手术、外伤、过敏史等 |
| 个人史 | 10% | 生活习惯、职业、居住地等 |
| 婚育史 | 10% | 婚姻状况、生育情况 |
| 家族史 | 10% | 家族遗传病史 |
| 体格检查 | 10% | 重要体征的记录 |
| 初步诊断 | 5% | 主要诊断的明确性 |

### 完整性评分标准

| 分数范围 | 说明 |
|----------|------|
| 90-100 | 病历要素齐全，记录详细完整 |
| 80-89 | 基本要素齐全，个别细节略缺 |
| 70-79 | 主要要素齐全，次要要素缺失 |
| 60-69 | 缺失 2-3 个主要要素 |
| <60 | 缺失 4 个或以上主要要素 |

### 常见问题

- **缺失主诉**: 没有明确的主要症状描述
- **现病史不完整**: 缺少起病诱因、症状演变过程
- **既往史遗漏**: 未记录重要既往疾病或手术史
- **体格检查缺失**: 只有"无异常"等笼统描述

## 2. 逻辑性 (Logicality)

逻辑性评估检查病历内容的医学逻辑和推理是否合理。

### 评估指标

| 指标 | 权重 | 检查内容 |
|------|------|----------|
| 症状-体征一致性 | 30% | 症状描述与体检发现是否吻合 |
| 检查-诊断逻辑 | 30% | 辅助检查是否支持诊断 |
| 治疗-诊断匹配 | 25% | 治疗措施是否针对诊断 |
| 时间逻辑 | 15% | 病程发展是否符合医学规律 |

### 逻辑性评分标准

| 分数范围 | 说明 |
|----------|------|
| 90-100 | 逻辑严密，推理过程清晰合理 |
| 80-89 | 逻辑基本清晰，个别细节存疑 |
| 70-79 | 逻辑基本合理，存在部分矛盾 |
| 60-69 | 存在明显逻辑问题，需要重新审视 |
| <60 | 逻辑混乱，诊断与证据矛盾 |

### 常见问题

- **症状与体征不符**: 主诉头痛但体检无神经系统阳性体征
- **检查与诊断矛盾**: 没有检查支持诊断或检查结果与诊断不符
- **治疗不合理**: 治疗措施与诊断不匹配
- **时间顺序混乱**: 病程描述时间顺序颠倒

## 3. 及时性 (Timeliness)

及时性评估检查病历记录的时效性，确保医疗行为可追溯。

### 评估指标

| 指标 | 权重 | 检查内容 |
|------|------|----------|
| 首次病程记录 | 35% | 入院后 8 小时内完成 |
| 医嘱执行时间 | 30% | 医嘱及时执行并记录 |
| 检查申请与结果 | 20% | 检查及时申请和结果记录 |
| 手术记录 | 15% | 术后 24 小时内完成 |

### 及时性评分标准

| 分数范围 | 说明 |
|----------|------|
| 90-100 | 所有记录均在规定时间内完成 |
| 80-89 | 大部分记录及时，个别延迟 |
| 70-79 | 存在 1-2 项记录延迟 |
| 60-69 | 存在 3 项或以上记录延迟 |
| <60 | 多项记录严重延迟或缺失 |

### 记录时间规范

| 记录类型 | 规定时间 | 说明 |
|----------|----------|------|
| 首次病程记录 | 入院后 8 小时 | 使用 LLM 评估时效性 |
| 住院记录 | 入院后 24 小时 | 完整住院病历 |
| 医嘱 | 即时 | 医生下达后及时执行 |
| 手术记录 | 术后 24 小时 | 由主刀医生签署 |
| 会诊记录 | 会诊后及时 | 通常 24 小时内 |

## 4. 规范性 (Normativity)

规范性评估检查病历书写的格式、术语和签名是否符合规范。

### 评估指标

| 指标 | 权重 | 检查内容 |
|------|------|----------|
| 格式规范 | 30% | 标题、段落、表格格式 |
| 术语使用 | 30% | 标准医学术语 |
| 签名完整 | 25% | 医生签名和时间 |
| 修改规范 | 15% | 修改是否符合规范 |

### 规范性评分标准

| 分数范围 | 说明 |
|----------|------|
| 90-100 | 完全符合规范，无格式问题 |
| 80-89 | 基本符合规范，个别细节问题 |
| 70-79 | 存在 2-3 处格式问题 |
| 60-69 | 存在 4-5 处格式问题 |
| <60 | 格式问题较多，影响可读性 |

### 常见问题

- **时间格式不统一**: 混用 24 小时制和 12 小时制
- **术语不规范**: 使用非标准医学术语或缩写
- **缺少签名**: 医生未签署姓名和时间
- **修改不规范**: 涂改、粘贴覆盖等

## 评分计算

### 维度评分计算

每个维度的最终得分基于发现的问题数量和严重程度：

```
维度得分 = 100 - ∑(问题对应分值)

问题分值：
- critical (严重): -30 分
- high (较重): -20 分
- medium (一般): -10 分
- low (轻微): -5 分
```

### 总体评分计算

```
总体得分 = Σ(维度得分 × 权重)

默认权重：
- 完整性: 25%
- 逻辑性: 25%
- 及时性: 25%
- 规范性: 25%
```

### 分数分级

| 分数范围 | 质量等级 | 说明 |
|----------|----------|------|
| 90-100 | 优秀 | 病历质量很高，基本无缺陷 |
| 80-89 | 良好 | 病历质量良好，有少量问题 |
| 70-79 | 合格 | 病历基本合格，需改进 |
| 60-69 | 基本合格 | 病历存在明显缺陷，需重点改进 |
| <60 | 不合格 | 病历质量差，存在严重问题 |

## 结果解读

### 问题列表

评估结果会生成问题列表，每条问题包含：

- **dimension**: 问题所属维度
- **severity**: 严重程度（critical/high/medium/low）
- **description**: 问题描述
- **recommendation**: 改进建议

### 优先级建议

1. **Critical 问题**: 立即处理，可能影响医疗安全
2. **High 问题**: 优先处理，影响医疗质量
3. **Medium 问题**: 计划处理，完善病历质量
4. **Low 问题**: 常规改进，提升规范性

## 自定义评估

### 自定义权重

如需调整评估维度权重，修改 `src/evaluator/evaluator.py`：

```python
evaluator = MedicalRecordEvaluator(
    weight_map={
        "completeness": 0.3,
        "logicality": 0.3,
        "timeliness": 0.2,
        "normativity": 0.2,
    }
)
```

### 自定义模板

使用高质量病历模板进行对比评估，发现差距：

1. 在 `src/templates/store.py` 中添加模板
2. 评估时指定 `template_id`
3. 查看差距分析和改进建议
