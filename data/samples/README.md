# 示例数据集说明

## 目录结构

```
data/samples/
├── normal/          正常病历（用于对比基准）
├── abnormal/        异常病历（包含各种质量问题）
└── templates/       优秀病历模板（用于相似度对比）
```

## 目录描述

### normal/ - 正常病历

存放符合医疗规范的正常病历，用于：
- 作为质控检测的基准样本
- 用于模板库的相似度对比
- 测试系统的正常检测流程

**文件说明：**
- `case_001.txt` - 高血压患者随访病历（慢性病管理）
- `case_002.txt` - 胃炎患者门诊病历（消化系统疾病）
- `case_003.txt` - 心律失常患者就诊病历（心血管系统）

### abnormal/ - 异常病历

存放存在质量问题的病历，用于测试异常检测功能：

**文件说明：**
- `abnormal_case_001.txt` - 缺项病历（缺少辅助检查、治疗方案不完整）
- `abnormal_case_002.txt` - 逻辑矛盾病历（既往史与检查结果矛盾）
- `abnormal_case_003.txt` - 时间线混乱病历（死亡记录与生命体征矛盾）

**质量问题类型：**
1. **缺项**：缺少必要的检查、诊断或治疗记录
2. **逻辑矛盾**：既往史与现病史、检查结果相互矛盾
3. **时间线错误**：病程记录时间逻辑混乱
4. **信息缺失**：关键信息（如药物过敏史）未填写
5. **重复记录**：相同内容重复记录

### templates/ - 优秀病历模板

存放结构化、规范化的病历模板，用于：
- **template_emergency.txt** - 急诊科病历模板（完整结构）
- **template_followup.txt** - 专科随访门诊模板（随访结构）

**模板用途：**
- 作为新病历书写的参考标准
- 用于相似度计算的基准模板
- 用于质控规则的验证

## 异常病历质量问题详解

### 1. 缺项问题（abnormal_case_001.txt）
- 未记录辅助检查具体结果
- 治疗方案不完整（未写明药物剂量、用法）
- 缺少出院医嘱
- 未记录患者转归

### 2. 逻辑矛盾问题（abnormal_case_002.txt）
- 既往史记录"无贫血"，但检查结果显示"Hb 85g/L（贫血）"
- 既往史记录"无高血压"，但家族史有高血压
- 精神分裂症病史与当前诊断失眠障碍的关联未说明

### 3. 时间线错误问题（abnormal_case_003.txt）
- 同一入院记录中重复记录相同事件
- 死亡记录与前面的生命体征记录矛盾
- 时间顺序混乱

## 使用建议

1. **测试异常检测功能**：
   - 将 normal/ 的病历作为"正常"参考
   - 将 abnormal/ 的病历作为"异常"样本
   - 验证系统能否正确识别质量问题

2. **测试模板相似度**：
   - 使用 templates/ 中的模板与样本病历对比
   - 验证相似度计算的准确性
   - 验证缺陷定位功能

3. **测试质控规则**：
   - 验证缺项是否能被规则检测发现
   - 验证逻辑矛盾是否能被规则检测发现
   - 验证时间线错误是否能被规则检测发现

## 数据使用规范

1. 本示例数据仅用于测试和演示目的
2. 病历内容均为模拟数据，不涉及真实患者信息
3. 请勿将示例数据用于临床诊断或医疗决策
4. 如需扩展测试数据，请保持相同的格式和规范
